# Contributing to Working Papers in Critical Search

## Submitting a Paper

See our full [submission guide](https://working-papers-in-critical-search.github.io/home/submit.html).

### Quick Version

1. Use our [paper template](https://github.com/Working-Papers-in-Critical-Search/paper-template) to create a repo in your GitHub account
2. Write your paper
3. Transfer the repo to our organization when ready
4. We review, you revise if needed
5. We tag a release â†’ Zenodo mints a DOI

### Not a Git User?

Email your materials to the editors. We'll handle the setup.

## Updating a Published Paper

Papers are living documents. To update:

1. Fork the paper's repo
2. Make your changes
3. Submit a pull request

Minor fixes get merged quickly. Major revisions may warrant a new version and DOI.

## Contributing to the Website

### Found a Bug?

[Open an issue](https://github.com/Working-Papers-in-Critical-Search/home/issues)

### Want to Improve Something?

1. Fork the `home` repo
2. Make your changes
3. Test locally: `quarto preview`
4. Submit a pull request

## Local Development

```bash
# Clone
git clone https://github.com/Working-Papers-in-Critical-Search/home.git
cd home

# Preview
quarto preview
```
